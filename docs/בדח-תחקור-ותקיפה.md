# בדח תחקור ותקיפה

## תיאור
- טופס לשמירת בדח תחקור ותקיפה במכשיר.
- כל בדח נשמר כרשומה נפרדת עם זמן עדכון אחרון.
- רשימת הבדחים ממוינת לפי זמן העדכון האחרון — המעודכן ביותר מופיע ראשון.
- הנתונים נשמרים מקומית בלבד ולא נשלחים החוצה.

## חוקי שמירה
- כל השדות אופציונליים — ניתן לשמור טופס ריק או חלקי.
- שדות אזימוט ידניים (גבולות גזרה ימני ושמאלי) — אופציונליים; אם מוזן ערך, חייב להיות מספר בטווח 0 עד 359.9.
- שדות בחירה (טוגל) מתחילים ללא בחירה — ניתן לשמור גם בלי בחירה, וניתן לבטל בחירה לאחר שבחרו.
- טעינת מטרה, טעינת מציין וטעינת עמדה אחורית אינם חובה לשמירה.
- שדות מטרה, שדות מציין ושדות עמדה אחורית שנטענו מרשומה שמורה ננעלים לעריכה ידנית.
- שדות מחושבים (אז' עמדה אחורית מטרה, אז' מציין מטרה, טווח ציון) מתמלאים אוטומטית כשמטרה והעמדה/מציין הרלוונטיים נטענו.
- ביצירת בדח חדש: העמדה האחורית נטענת אוטומטית מהעמדה הנוכחית אם קיימת; ניתן להחליף לעמדה אחרת מהמאגר או לנקות (כמו טעינת מטרה). אין חובה לטעון עמדה לפני שמירה.
- בעריכת בדח שמור: פרטי העמדה האחורית נשארים כפי שנשמרו — לא מתעדכנים מהעמדה הנוכחית.

## תנאים
- **מסלול קשתי = כן** — מוצג שדה כיוון קשת (אופציונלי).
- **הצבה ידנית = כן** — מוצגים שדות הגבהה וצידוד; מוצגים רק כשבוחרים כן.
- **הצבה ידנית = לא** — הגבהה וצידוד מוסתרים.

## שמירה מקומית
- הרשומות נשמרות במכשיר ונשארות לאחר סגירה ופתיחה מחדש של האפליקציה.
- ממסך רשימת הבדחים: לחיצה ארוכה על כרטיס פותחת תפריט; משם ניתן למחוק. המחיקה דורשת אישור, מסירה את הרשומה מהמכשיר ולא ניתן לשחזר מתוך האפליקציה.
